        "**1. What is a RESTful API?**\n",
        "\n",
        "A RESTful API qq(Representational State Transfer Application Programming Interface) is a way for two computer systems to communicate over the internet using standard HTTP methods. It follows the principles of the REST architectural style, which makes web services simple, scalable, and stateless.\n",
        "Key features of a RESTful API:\n",
        "\n",
        "1. Stateless – Each request from a client must contain all the information needed to process it. The server does not store session data between requests.\n",
        "\n",
        "2. Resource-based – Everything is treated as a resource (e.g., users, products, posts) and is identified by a unique URL.\n",
        "\n",
        "3. Standard HTTP Methods – RESTful APIs typically use:\n",
        "\n",
        "GET → Retrieve data\n",
        "\n",
        "POST → Create new data\n",
        "\n",
        "PUT/PATCH → Update existing data\n",
        "\n",
        "DELETE → Remove data\n",
        "\n",
        "4. Data formats – Commonly uses JSON or XML for data exchange (JSON is more popular)."
      ]

        "**8. What is the difference between GET and POST HTTP methods?**\n",
        "\n",
        "- Difference between GET and POST HTTP Methods\n",
        "\n",
        "**GET Method**\n",
        "\n",
        "- Used to request data from the server.\n",
        "\n",
        "- Sends data as URL parameters (e.g., example.com/page?name=John).\n",
        "\n",
        "- Data is visible in the URL.\n",
        "\n",
        "- Typically used for reading/fetching information.\n",
        "\n",
        "- Safe and idempotent → repeating the same GET request doesn’t change data.\n",
        "\n",
        "**POST Method**\n",
        "\n",
        "- Used to send data to the server (to create or update resources).\n",
        "\n",
        "- Sends data in the request body, not the URL.\n",
        "\n",
        "- Data is not visible in the URL.\n",
        "\n",
        "- Typically used for form submissions, login, file uploads.\n",
        "\n",
        "- Not idempotent → repeating the same POST request may create duplicate records"
      ]

        "Standardization → Developers understand the API easily.\n",
